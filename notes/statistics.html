Descriptive Statistics

Central Tendency — Mean, Median, Mode
Step 1 Always
Measure Formula Kab Use Karo Avoid When
Mean (Average) Σx / n Normal data, outliers nahi hain Outliers ho toh misleading hoga
Median Middle value (sorted) Skewed data, outliers hain (salary, income) Categorical data
Mode Sabse zyada repeat value Categorical data (product category, city name) Numerical continuous data
Income data mein hamesha Median use karo — ek billionaire bana deta hai Mean ko useless! (Bill Gates effect)
Mean > Median = Right skewed (outliers right side). Mean < Median = Left skewed. Symmetric data mein Mean ≈ Median ≈ Mode
Spread / Variability — Variance, SD, IQR
Spread Measure
Variance (Population)
σ² = Σ(x - μ)² / N
Variance (Sample)
s² = Σ(x - x̄)² / (n-1)
Standard Deviation
σ = √Variance
IQR (Interquartile Range)
IQR = Q3 - Q1
Standard Deviation Use Karo
Normal data mein spread batane ke liye. "Avg se kitna door?" Low SD = data tightly packed, High SD = data spread out.
IQR Use Karo
Outlier detect karne ke liye. Outlier = value < Q1 - 1.5×IQR ya > Q3 + 1.5×IQR (Box Plot method)
Sample variance mein n-1 (not n) kyun? — Bessel's Correction. Sample se population estimate karte ho toh divide by (n-1) se bias kam hota hai.
CV (Coefficient of Variation) = (SD / Mean) × 100 — Different units waale datasets compare karne ke liye. Stock A ka CV vs Stock B ka CV.
Shape of Distribution — Skewness & Kurtosis
Distribution Shape
MetricFormula (approx)Value Interpret karo
Skewness Σ(x-x̄)³ / (n × σ³) 0 = Symmetric | +ve = Right skewed | -ve = Left skewed
Kurtosis Σ(x-x̄)⁴ / (n × σ⁴) - 3 0 = Normal | +ve = Heavy tails (Leptokurtic) | -ve = Thin tails (Platykurtic)
Financial data (stock returns) mein Kurtosis zyada hota hai → Fat tails → Crash ka risk zyada. Normal assume karna dangerous hai!

Probability

Basic Rules
Addition Rule (OR)
P(A∪B) = P(A) + P(B) - P(A∩B)
Multiplication Rule (AND)
P(A∩B) = P(A) × P(B|A)
Complement
P(A') = 1 - P(A)
Independent events mein P(A∩B) = P(A) × P(B)
Bayes' Theorem
Key Formula
P(A|B) = [P(B|A) × P(A)] / P(B)
Kab use karo Spam filter, Medical diagnosis, A/B testing, Recommendation systems. "Given ki ye email spam hai, to sender fraud hai ki probability?"
Prior × Likelihood = Posterior. New evidence aane pe belief update karo — Bayesian Thinking!
Conditional Probability
P(A|B) = P(A∩B) / P(B)
Example
Jo customer email open karta hai (B), uske purchase karne ki probability (A|B)?
Data Analytics Use
Funnel analysis, churn prediction, feature importance in ML models

Probability Distributions

Normal Distribution (Gaussian)
Most Important
PDF Formula
f(x) = (1 / σ√2π) × e^[-(x-μ)² / 2σ²]
68-95-99.7 Rule (Empirical Rule)
μ ± 1σ → 68% data
μ ± 2σ → 95% data
μ ± 3σ → 99.7% data
Z-Score Formula
Z = (x - μ) / σ

"Value kitna SDs away hai mean se?"
Z-score > 3 ya < -3 = Outlier confirm. Z-table se area/probability nikalo.
Heights, test scores, measurement errors → Normal Distribution. Real life mein CLT ki wajah se bahut sari cheezein normal ho jaati hain!
Binomial Distribution
P(X=k) = C(n,k) × pᵏ × (1-p)ⁿ⁻ᵏ
Kab use karo Fixed n trials, har trial success/fail, same probability. Email CTR, A/B test conversions, Quality control (defective/non-defective)
Mean = np | Variance = np(1-p)
Poisson Distribution
P(X=k) = (λᵏ × e⁻λ) / k!
Kab use karo Fixed time/space mein events count. Website visits per hour, customer arrivals per day, errors per 1000 lines of code
Mean = λ | Variance = λ
Central Limit Theorem (CLT) — Sabse Powerful Concept
🔑 Key Theorem
Kya bolti hai CLT Agar koi bhi distribution ho (skewed bhi), uske samples ke means ki distribution NORMAL hogi — agar n ≥ 30 ho.
Sample Mean Distribution: x̄ ~ N(μ, σ²/n) | Standard Error = σ/√n
CLT ki wajah se hi Z-test, t-test, confidence intervals kaam karte hain. Yahi stats ka foundation hai!

Inferential Statistics

Confidence Intervals
Estimation
Z-Interval (n≥30 or σ known)
CI = x̄ ± Z* × (σ/√n)

90% → Z* = 1.645
95% → Z* = 1.96
99% → Z* = 2.576
t-Interval (n<30 or σ unknown)
CI = x̄ ± t* × (s/√n)

t* value = t-table se
df = n - 1
95% CI ka matlab: "Agar 100 baar sample lo, to 95 baar interval true mean contain karega." Not: "95% chance ki mean is range mein hai."
Wider CI = Less confident exact value. Narrow CI = More precise (needs larger n). CI narrow karna ho → n badhao!
Sample Size Calculation
For Mean
n = (Z* × σ / E)²   where E = margin of error
For Proportion
n = (Z*)² × p(1-p) / E²   (p unknown ho to p=0.5 use karo → max sample size)
A/B test se pehle sample size calculate karo — isko "Power Analysis" kehte hain. Varna false result aa sakta hai.

Hypothesis Testing

Framework — Step by Step
Process
Step 1: H₀ (Null) aur H₁ (Alternative) define karo
→ H₀: "No effect" (status quo). H₁: "Effect hai" (jo tum prove karna chahte ho)
Step 2: Significance Level (α) decide karo
→ Usually α = 0.05 (5%). Medical fields mein 0.01
Step 3: Sahi test choose karo
→ (Neeche table dekho)
Step 4: Test statistic calculate karo & p-value nikalo
→ p-value = "Agar H₀ true ho to ye data milne ki probability"
Step 5: Decision
→ p < α : Reject H₀ (significant result!)
→ p ≥ α : Fail to Reject H₀ (not significant)
Kaunsa Test Kab Use Karo
SituationTestFormula
1 sample mean vs known value (n≥30) Z-Test Z = (x̄ - μ₀) / (σ/√n)
1 sample mean vs known value (n<30) One-Sample t-Test t = (x̄ - μ₀) / (s/√n)
2 independent group means compare Two-Sample t-Test t = (x̄₁ - x̄₂) / SE
Before/After same group Paired t-Test t = d̄ / (Sd/√n)
3+ groups means compare ANOVA (F-Test) F = MSB / MSW
Proportions compare Chi-Square / Z-prop χ² = Σ(O-E)²/E
Non-normal data, 2 groups Mann-Whitney U Rank-based test
Type I Error (α) = False Positive — "H₀ true thi par reject kar diya" | Type II Error (β) = False Negative — "H₁ true thi par reject nahi kiya". Power = 1 - β
p-value < 0.05 ka matlab practical significance nahi hai! Large n ke saath trivial difference bhi significant ho jaata hai. Effect size bhi dekho (Cohen's d).
Chi-Square Test — Categorical Data
χ² = Σ [(Observed - Expected)² / Expected]
Goodness of Fit Test
Observed frequencies expected distribution se match karti hain? (Ek variable)
Test of Independence
Kya do categorical variables independent hain? (Contingency table). Gender vs Product Preference
Expected = (Row Total × Column Total) / Grand Total
Chi-square ke liye: Expected frequency > 5 honi chahiye har cell mein. Chhoti frequency → Fisher's Exact Test use karo.

Correlation & Association

Pearson Correlation (r)
r = Σ[(x-x̄)(y-ȳ)] / √[Σ(x-x̄)² × Σ(y-ȳ)²]
Kab use karo Linear relationship, Normal data, Continuous variables. -1 to +1 range.
Interpretation
|r| > 0.7 → Strong | 0.4-0.7 → Moderate | <0.4 → Weak
Spearman Rank (ρ)
ρ = 1 - (6 × Σd²) / (n × (n²-1))
d = rank difference
Kab use karo Non-linear relationship, Ordinal data, Outliers hain. Non-parametric version of Pearson.
Customer satisfaction rank vs sales rank → Spearman
⚠️ Correlation vs Causation
Critical Concept
"Ice cream sales aur drowning incidents ka r = +0.85" — Dono summer mein badhte hain (confounding variable: season). High correlation ≠ Causation. Causation prove karne ke liye → Controlled Experiment / RCT chahiye.
Spurious Correlation
Nicolas Cage movies aur pool drowning deaths ka perfect correlation hai — coincidence!
Confounding Variable
Hidden third variable jo dono ko affect karta hai. Always check before claiming causation.

Regression Analysis

Simple Linear Regression
Prediction
Model
ŷ = β₀ + β₁x + ε
Slope (β₁)
β₁ = Σ(x-x̄)(y-ȳ) / Σ(x-x̄)²
= r × (Sy/Sx)
Intercept (β₀)
β₀ = ȳ - β₁ × x̄
R² (R-Squared)
Model kitna variation explain karta hai. R²=0.85 → 85% variation explained. Higher = Better fit.
Residuals
e = y - ŷ (actual - predicted). Residuals random hone chahiye. Pattern = problem hai!
Adjusted R² use karo multiple regression mein — extra variables add karne se plain R² artificially badhta hai.
Regression Assumptions (LINE)
AssumptionKya Check KaroViolation Fix
LinearityScatter plotTransform (log, sqrt)
IndependenceResidual plotTime series — add lag variables
Normality of ErrorsQ-Q Plot, Shapiro-WilkTransform Y variable
Equal Variance (Homoscedasticity)Residuals vs Fitted plotWeighted regression
No MulticollinearityVIF < 5 (ideally <10)Remove correlated predictors
Overfitting detect karna: Training accuracy ≫ Test accuracy → Overfit. Solution: Cross-validation, Regularization (Ridge/Lasso)

⚡ Top 10 Pro Tips & Tricks
🔍 EDA Pehle Karo
Koi bhi analysis se pehle Exploratory Data Analysis. Visualize first, then test.
📐 Normality Check
Parametric test (t-test, ANOVA) se pehle normality check karo — Shapiro-Wilk ya Q-Q Plot.
🎯 Effect Size
p-value ke saath Cohen's d (mean diff) ya Cramér's V (chi-sq) bhi report karo.
Cohen's d = (μ₁-μ₂) / σ_pooled
🔄 Multiple Testing
Multiple tests karo to false positive badhta hai. Bonferroni Correction use karo.
α_adjusted = α / k
📊 Log Transform
Right-skewed data (salary, revenue, price) → log(x) transform → Normal ho jaata hai.
🚨 Outlier Policy
Remove karne se pehle reason dhundho. Business logic se decide karo — blindly remove mat karo.
✅ Cross-Validation
Model evaluate karo k-Fold CV se. Overfitting detect karne ka best way.
🎲 Power Analysis
A/B test shuru karne se pehle minimum sample size calculate karo (80% power standard hai).
📉 Residual Plot
Regression ke baad residuals zaroor plot karo — pattern dikha to model wrong hai!
🧠 CLT Remember
n≥30 hote hi sample mean Normal ho jaata hai — yahi sab inference ka base hai!
